# Modifiers

A `modifier` is an object (middleware or plugin) that modifies the `x` and `y`
coordinates in some way, or provides useful data to the consumer. Popper
provides the most useful modifiers in its core already for you to import.
Anything you don't use will be tree-shaken away.

## Example

A very basic modifier that shifts the coordinates by 1 pixel:

```js
const shiftBy1Px = {
  // The name of the modifier.
  name: 'shiftBy1Px',
  // The main logic which returns new coordinates.
  fn({ x, y }) {
    return {
      x: x + 1,
      y: y + 1,
      data: {
        // Add custom data to be read by other modifiers or the consumer.
        // The `name` property of the modifier identifies where this data goes
        // in the main `modifiersData` object.
      },
    };
  },
};
```

All properties in the returned object of `fn` are optional, but an object must
be returned (not `void`).

### Function

Popper's core modifiers are factory functions, which allows you to pass options
to configure their behavior:

```js
const shiftByAmount = (amount = 0) => ({
  name: 'shiftByAmount',
  fn: ({ x, y }) => ({ x: x + amount, y: y + amount }),
});
```

## Type

```flow
type Modifier = {
  name: string,
  fn: (
    modifierArguments: ModifierArguments
  ) =>
    | $Shape<{| ...Coords, data: Obj |}>
    | Promise<$Shape<{| ...Coords, data: Obj |}>>,
};

type ModifierArguments = {|
  // Positioning coordinates.
  x: number,
  y: number,
  // Stateful placement, which can be changed by other modifiers, e.g. `flip`.
  placement: Placement,
  // Stateless placement, passed in as an option to `computePosition()`.
  initialPlacement: Placement,
  // The reference and popper `Rect`s, an object of shape {width, height, x, y}.
  rects: ElementRects,
  // The reference and popper elements themselves.
  elements: Elements,
  // Data from other modifiers.
  modifiersData: Obj,
  // Platform object containing methods.
  platform: Platform,
  // Type of CSS position property used.
  strategy: PositioningStrategy,
  // Reset the modifiers lifecycle with a new placement.
  scheduleReset: ({|
    placement: Placement,
  |}) => void,
|};
```
