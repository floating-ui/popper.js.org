# autoPlacement

The `autoPlacement` modifier automatically chooses the best placement for the
popper, i.e. the one with most space.

## Usage

```js
import { computePosition, autoPlacement } from '@popperjs/dom';

computePosition(reference, popper, {
  modifiers: [autoPlacement()],
});
```

## Options

`autoPlacement()` takes an object of options:

```flow
type Options = {
  variation: null | 'start' | 'end',
  crossAxis: boolean,
  allowedPlacements: Array<Placement>,
  autoVariation: boolean,
  ...DetectOverflowOptions,
};
```

### `variation`

Describes whether to use a `variation` placement as the preferred placement.

```js
autoPlacement({
  variation: 'end', // 'top-end', 'bottom-end', 'right-end', 'left-end'
});
```

### `crossAxis`

Describes whether to check for overflow on the cross axis in addition to the
main axis when choosing the ideal placement.

```js
autoPlacement({
  crossAxis: true, // false by default
});
```

### `allowedPlacements`

Describes which placements are allowed to be chosen.

```js
autoPlacement({
  // Won't choose 'left' or 'right'
  allowedPlacements: ['top', 'bottom'],
});
```

### `autoVariation`

Describes whether to automatically choose the best variation if a `variation` is
specified.

```js
autoPlacement({
  variation: 'start',
  // Will also choose `end` placements if those fit better
  autoVariation: true,
});
```
